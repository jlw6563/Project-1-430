Pokedex API Documentation

Get All Pokemon Names

Relative URL: /PokemonNames

Supported Methods: GET, HEAD

Query Params: None

Returns: Array of PokemonName objects. Example: [{ name }]

Example Usage: GET /PokemonNames

Get Pokemon Names With Types

Relative URL: /PokemonTypes

Supported Methods: GET, HEAD

Query Params: type (string, optional) — filter by type. Example: type=fire & weakness (string, optional) — filter by weakness. Example: weakness=water

Returns: Array of PokemonType objects. Example: [{ name, type }]

Returns: 400 Bad Request with message and id=NothingFound Bad Request Nothing Found.

Example Usage: GET /PokemonTypes & GET /PokemonTypes?type=fire & GET /PokemonTypes?weakness=water & GET /PokemonTypes?type=fire&weakness=water

Get All Pokemon

Relative URL: /AllPokemon

Supported Methods: GET, HEAD

Query Params: type (string, optional) — filter by type & weakness (string, optional) — filter by weakness.

Returns: 400 Bad Request with message and id=NothingFound Bad Request Nothing Found.

Returns: 400 Bad Request with message and id= id: 'NothingFound' when required fields are missing.

Example Usage: GET /AllPokemon & GET /AllPokemon?type=grass & GET /AllPokemon?weakness=ice & GET /AllPokemon?type=grass&weakness=ice

Get All Caught Pokemon

Relative URL: /AllCaught

Supported Methods: GET, HEAD

Query Params: type (string, optional) — additional filter by type applied after caught filter & weakness (string, optional) — additional filter by weakness applied after caught filter.

Returns: Array of PokemonFull objects where caught === true.

Returns: 400 Bad Request with message and id=NothingFound Bad Request Nothing Found.

Example Usage: GET /AllCaught & GET /AllCaught?type=electric

Add or Update a Pokemon

Relative URL: /AddPokemon

Supported Methods: POST

Body Params: num (number, required) — Pokedex number; updates existing if present & name (string, required) — Pokemon name & height (string or number, required) — height value & weight (string or number, required) — weight value.

Returns: 201 Created with message when a new record is added.

Returns: 204 No Content with message when an existing record is updated.

Returns: 400 Bad Request with message and id=missingParams when required fields are missing.

Example Usage: POST /AddPokemon

Mark Pokemon As Caught

Relative URL: /Caught

Supported Methods: POST

Body Params: caught (string, required) — Pokemon name to mark as caught.

Returns: 204 No Content with message when the named pokemon is marked as caught.

Returns: 400 Bad Request with message and id=PokemonNotFound when name is missing or not found.

Example Usage: POST /Caught
